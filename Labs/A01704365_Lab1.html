Investigación

URL: es una abreviación de Uniform Resocurce Locator, es básicamente una dirección a la cual puedes acudir para encontrar cierta información en el internet
MIME: significa Mulitpurpose Internet Mail Extensions, y son una serie de características que se implementan a el uso del servicio de correo electrónico, como por ejemplo la capacidad de cambiar de fuente o de color de texto.[1]
HTML5: ( Hyper Text Markup Language) Se trata de la última versión del lenguaje HTML, que incluye diferentes tecnologías para facilitar la creación de páginas web con su uso. Algunas de estas tecnologías se clasifican en multimedia, conectividad, semántica, gráficos, etc.[2]
JS: Es un lenguaje de programación orientado a la creación de páginas web, aunque también es usado en otros ámbitos, que soporta múltiples paradigmas de programación.
PHP: Es un lenguaje de programación ampliamente usado en el ámbito del desarrollo web y puede ser usado junto con HTML. El código de php es generado en el servidor.
JSP: Significa Java Server Pages y como otras en esta lista, es una tecnología disañada para el diseño y desarrollo de paginas web usando Java. Las páginas que usan esta tecnología están escritas en HTML o en XML adecuadas para formar scripts adecuados para un servidor [3]
AJAX: Ajax es una manera de llamarle al uso de tecnologías específicas simultáneamente, y significa Asynchronous JavaScript and XML . Con el conjunto de estas tecnologías es que se crean páginas rápidas y dinámicas con los usuarios, al implementar JavaScript. [4]
Métodos y códigos de errores HTTP: Si quieres hacer cualquier petición a algún servidor mediante HTTP necesitas hacerlo con alguno de los métodos que le dicen al servidor las diferentes posibles acciones a tomar. Algunos ejemplos de estos métodos son: GET, DELETE, CONNECT. Junto con estos métodos van los errores que pueden generar al no poder realizar con éxito las órdenes. Algunos de éstos son: 404 not found o 400 bad request(para el GET), 0 204 (en caso que el PUT no devuelva ningún contenido)

Preguntas a responder

¿Cuál es la diferencia entre el internet y la World Wide Web?

La World Wide Web en sí es un sistema que contiene diversos documentos de hipermedia, o hipertexto. Estos documentos contenidos en la WWW pueden ser accesados desde el internet. El internet, por su parte es una red usada para transimit informacion, como por ejemplo los documentos de la WWW.[6] Inernet es el medio.

¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?

El método HTTP de GET se usa cuando quieres adquirir un recurso dentro de un servidor web, en caso de no encontrar el recurso, salta un error 404 not found. El tipo de recurso que puede detectar varía desde un HTML hasta imágenes. HEAD no regresa el recurso en sí, si no que únicamente regresa su información. POST se usa cuando es necesario enviar algún tipo de informacion al servidor web. El metodo PUT inserta o genera un recurso nuevo dentro del servidor, o en caso de que existiera uno con el mismo nombre, se encarga de sustituirlo. El DELETE elimina el recurso especificado, y finalmente el método PATCH hace modificaciones parciales a los elementos desados dentro del servidor. [7][8]

¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué?

Por defecto se usa el método GET[11], pero se recomienda el método POST, ya que éste se usa cunado tu quieres enviar cualquier tipo de dinformación al servidor.

¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página a través de un URL?

Se utiliza el método GET.

Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?

No se trata de un error, si no que es una confirmación de que el método GET respondió de manera positiva en la búsqueda del recurso que buscabas.

¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?

Si, ya que se trata de un error en el método GET, que intenta recibir información de un recurso que no se encuentra en la dirección indicada, provocando el error 404 (not found).

¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?

El error 500 sale cuando hay un error que el servidor no puede especificar, pero que inpide su funcionamiento, por lo que si es responsabilidad del desarrollador, y no del navegante.

¿Qué significa que un atributo HTML5 esté depreciado o desaprobado (deprecated)? Menciona algunos elementos de HTML4 que en HTML5 estén desaprobados.

Éstos son atributos que están en HTML pero que su uso no es 100% recomendado. Esto sucede generalente cuando se hacen actualizaciones la lenguaje; algunos de los atributos quedan obsoletos al ser implementados unos nuevos. Unos ejemplos de éstos, que eran aprobados en HTML4 pero ya no lo son: basefont, font, o big.(en genreal los que dan formato a la página, ya que en HTML5, esta funcion se separó).

¿Cuáles son las diferencias principales entre HTML 4 y HTML5?

Al ser HTML5 una nueva version de HTML4, una de sus diferencias es que HTML5 cunata con mas soporte en navegadores como Chrome, FireFox u Opera, sin mencionar navegadores móviles. De igual manera, HTML4 tiene mas etiquetas que su antecesor, como por ejemplo audio, video o canvas. Así como HTML5 tiene soporte a elementos multimedia por defecto, cosa que HTML4 no tenía. En general, entre las versiones avanzaron se fue haciendo más y más simple, con un mejor manejo y detección de errores.[9] Ademas de esto, en HTML4 se solían incluir los elementos que daban formato a la página, mientras que en HTML5 se dedica únicamente al contenido.

¿Qué componentes de estructura y estilo tiene una tabla?

Las tablas tienen dos componentes principales, las filas (rows), y titulos de columnas (Colmn headers). Dentro de estas puedes colocar contenido mediante las etiquetas td. Tambipen puedes editar el estilo de estos elementos mediante atributos como: colspan para unir columnas o rowspan para unir dos filas.

Sobre el ciclo de vida y desarrollo de los sistemas de informacion

¿Cuál es el ciclo de vida de los sistemas de información?

Es un ciclo que consta de siete diferentes fases:

  1. Planificacion: definición de alcance, justificación y elección de metodología. Así como la definición de lapsos temporales y asignación de roles
  2. Análisis: definición de los diferetes requisitos de Software
  3. Diseño: selección de los requisitos aplicables y evaluación del software y hardware que se usará en el proyecto. Se determina cómo se cumpliran con los requisitos del cliente
  4. Desarrollo: es el inicio de la produccion del sistema de software
  5. Integración y pruebas: verificar mediante pruebas la detecicón y correción de errores, y verfificar que el diseño anteriormente planeteado soluciona los problemas que se busca solucionar
  6. Implementación: se refiera a la instalacion del software y hardware, pruebas finales y la capacitación de los usuarios, ya entra en funcionamiento el sistema
  7. Mantenimiento: la implementación de ajustes que sean necesarios para que sea mejorado el rendimiento y funcionalidad del sistema una vez en funcionamiento. Así mismo, se encarda de la corrección de errores.
[10]
